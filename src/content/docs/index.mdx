# Building. Breaking. Learning. Documenting.

This site is my personal engineering notebook.

Everything here exists because I either built it, broke it, fixed it, or wanted
to remember how it worked six months later.

The goal is simple:

- Build a better homelab
- Automate repetitive work
- Learn cloud-native technologies
- Document everything worth remembering

---

## Philosophy

> If I have to solve the same problem twice,
> it deserves documentation.

Most articles here are written for future me.

If they save someone else hours of debugging,
that's a nice side effect.

:::tip

This documentation changes frequently as I learn new technologies and improve my infrastructure.

Expect mistakes, revisions, and plenty of experiments.

:::
