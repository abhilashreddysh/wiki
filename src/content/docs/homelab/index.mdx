This section describes my personal homelab:

> a self-hosted environment used for learning, experimentation, and running real services with real constraints.

It is designed to be rebuildable from bare metal

---

## What This Homelab Runs

- Virtualization and container orchestration
- Internal services (DNS, monitoring, dashboards)
- CI/CD experiments and automation
- Storage, backups, and media workloads

This is not a “try everything once” setup.  
Anything unstable gets documented, broken, fixed, or removed.

## Design Principles

    1. Git is the source of truth.

    2. Backups are useless unless restored.

    3. Security beats convenience.
